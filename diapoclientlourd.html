Diaporama - Projet Client Lourd

1. Ressources numériques

Projet codé en Python avec base de données SQLite. Structure modulaire avec des fichiers pour les tickets, utilisateurs, équipements.

2. Normes et standards

Organisation du code respectant le principe de la POO et la séparation des responsabilités. Usage de standards Python (sqlite3, fichiers modules).

3. Habilitations

Gestion des utilisateurs avec rôles (admin, user, technicien). Accès restreint aux fonctions sensibles selon le profil connecté.

4. Continuité de service

La base SQLite peut être sauvegardée manuellement ou via un script. Structure simple, mais efficace pour un usage local.

5. Respect des règles d'utilisation

Fonctions accessibles selon les droits. Validation des entrées et gestion des erreurs utilisateur prévues.

6. Gestion des incidents

Système de tickets pour suivre les incidents, affectation possible à un équipement ou un technicien, et changement d'état (ouvert, en cours, résolu).

7. Tests d’intégration

Des tests manuels ont été réalisés pour valider le fonctionnement de chaque fonctionnalité du projet.

8. Déploiement

L'application peut être exécutée via Python ou convertie en .exe pour une installation client lourd. Aucun serveur requis.
